### 🛣️ Comment puis-je contribuer ?

#### Signaler des Bugs 🐛
* Vérifiez l'onglet [Issues](https://github.com/Artleboss2/Frython/issues) pour voir si le bug a déjà été signalé.
* Si ce n'est pas le cas, ouvrez une nouvelle issue en utilisant le modèle **Bug Report**.
* Soyez aussi précis que possible : étapes pour reproduire, version de Python et logs d'erreurs.

#### Suggérer des Améliorations ✨
* Ouvrez une issue avec le tag `enhancement`.
* Expliquez pourquoi cette fonctionnalité serait utile et comment elle devrait fonctionner.

#### Pull Requests (PRs) 🛠️
1.  **Forkez** le dépôt.
2.  **Clonez** votre fork : `git clone https://github.com/VOTRE_USERNAME/Frython.git`.
3.  **Créez une branche** pour votre fonctionnalité : `git checkout -b feature/NomDeLaFonctionnalite`.
4.  **Commitez** vos changements : `git commit -m 'Ajout de la fonctionnalité X'`.
5.  **Poussez** vers la branche : `git push origin feature/NomDeLaFonctionnalite`.
6.  **Ouvrez une Pull Request** vers la branche `main`.

---

### 🎨 Guide de Style & Standards
* **Style de Code :** Suivez les conventions de nommage [PEP 8](https://www.python.org/dev/peps/pep-0008/).
* **Documentation :** Assurez-vous que chaque nouvelle fonction possède une docstring claire.
* **Messages de Commit :** Utilisez des titres descriptifs (ex: `fix: résolution de l'erreur de récursion` au lieu de `fix stuff`).
